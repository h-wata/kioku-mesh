# v0.3.3 Release Notes

A documentation release for the OSS launch. No code or on-disk storage changes —
upgrading is optional and changes nothing at runtime.

## Highlights

- **README rebuilt to lead with what makes kioku-mesh different** (PR #136, #137).
  The pitch is now "shared memory for AI coding agents, across tools and machines"
  — a peer-to-peer mesh you run yourself (LAN / VPN / Tailscale), readable by
  Claude Code, Codex CLI, Gemini CLI, and any other MCP client — rather than
  generic "persistent memory". A new "Why kioku-mesh" section frames the
  cross-machine context-fragmentation problem.
- **Visual walkthrough.** The title is now the project logo, and a 20-second demo
  GIF shows one agent saving a decision and an agent on another host recalling it
  live over the mesh.
- **Architecture made legible.** The Multi-Host Mesh section gains two Mermaid
  diagrams: a data-flow view (agents → zenohd source of truth → rebuilt SQLite
  read index, replicated host-to-host) and a hub-and-spoke topology view.
- **Sharper mental model.** Mesh mode is documented as source-of-truth + local
  read cache, the trusted-network security model is restated, and mTLS peer
  authentication is noted as under consideration.

## Upgrade

No code changes — nothing to do at runtime. To pull the refreshed package metadata:

```bash
uv tool install --upgrade git+https://github.com/h-wata/kioku-mesh.git
# or, from PyPI:
pip install --upgrade kioku-mesh
```
