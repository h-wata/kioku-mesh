# v0.3.0 Release Notes — v0.3 First-Impression Overhaul

## Highlights

- **Tier 0: Local backend** (#109) — `mesh-mem init --mode local` works with no `zenohd` on PATH. Single-machine persistent memory in 30 seconds.
- **Tier 1: Embedded zenoh router** (#112, integration fix #120) — `mesh-mem mesh start` opens an in-process zenoh router (no binary install). Multi-host ephemeral mesh without `zenohd`.
- **README v0.3 overhaul** (#110) — hero → Tier 0/1/2 quickstart → Power users narrative.
- **MemoryBackend abstraction** (#109) — unified `get_backend()` surface shared by CLI, MCP server, and all Tiers.
- **`mesh-mem doctor`** (#84) — first-touch diagnostic with embedded router status.
- **`mesh-mem mcp install`** (#85) — one-shot Claude Code / Codex CLI MCP registration.
- **Forward-compat for Observation schema** (#75) — older readers preserve unknown fields via `_extras`.

## New Commands

| Command | Description |
|---------|-------------|
| `mesh-mem init --mode local` | Initialize local (Tier 0) backend — no zenohd needed |
| `mesh-mem mesh start [--listen tcp/0.0.0.0:17447]` | Start in-process zenoh router (Tier 1) |
| `mesh-mem mesh join <peer>` | Connect to a peer endpoint and verify connectivity |
| `mesh-mem doctor` | Run diagnostic checks (zenohd, config, state dir, embedded router) |
| `mesh-mem mcp install --client <claude-code\|codex-cli>` | Register MCP server with one command |
| `mesh-mem init --mode localhost\|hub\|spoke\|local` | Generate zenohd config or local config |

## Breaking Changes

- **`mesh-mem delete` cursor behavior changed** (#66): bulk-delete now pages past `MAX_SEARCH=10000` using a strict `(created_at, observation_id)` DESC cursor. Scripts relying on the 10k hard-abort behavior will now complete fully.
- **CLI/MCP strings are now English-only** (#53): previously mixed Japanese/English output strings are now uniformly English. Scripts grepping Japanese substrings (e.g. `保存完了`, `削除`) from CLI or MCP output will break.
- **`memory_type` validated against closed enum** (#84 follow-up): `note / decision / bug / pattern / config / summary` only. Old aliases (`bugfix`, `discovery`, `fact`, `status`, `learning`) are rejected at write time.

## Migration Guide (v0.2.x → v0.3.0)

### Upgrade steps

```bash
# If installed via uv tool:
uv tool install --upgrade git+https://github.com/h-wata/mesh-mem.git

# If installed via pip (editable):
pip install -e ".[dev]"
```

### New Tier 0 quick start (no zenohd)

```bash
mesh-mem init --mode local
mesh-mem save "my first memory"
mesh-mem search "memory"
```

### Tier 1 multi-host (no zenohd binary)

Host A:
```bash
mesh-mem mesh start --listen tcp/0.0.0.0:17447
```

Host B:
```bash
ZENOH_CONNECT=tcp/<host-a-ip>:17447 mesh-mem save "cross-host memory"
```

### Backend selection

| Method | Backend |
|--------|---------|
| `MESH_MEM_BACKEND=local` | Tier 0 SQLite only |
| `ZENOH_CONNECT=...` (default) | Tier 1/2 Zenoh |
| `mesh-mem init --mode local` | writes `~/.config/mesh-mem/config.yaml` with `backend: local` |

### Config file changes

`~/.config/mesh-mem/config.yaml` (new in v0.3):
```yaml
backend: local  # Use Tier 0 SQLite-only mode
```
